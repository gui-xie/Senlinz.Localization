Overview

Documentation that feels like a real developer portal.

Senlinz.Localization is a JSON-driven localization source generator for .NET. It turns localization keys into strongly typed APIs, supports culture-specific resources, and keeps fallback behavior predictable.

Strongly typed accessors Culture-aware resources Lightweight runtime contracts
Why teams use it

Keep localization close to code

  • Generate L members directly from l.json.
  • Override only the culture-specific values that actually change.
  • Fall back to default JSON text when resources or keys are missing.
5-minute setup 
dotnet add package Senlinz.Localization

<ItemGroup>
  <AdditionalFiles Include="l.json" />
  <None Update="l.json" CopyToOutputDirectory="PreserveNewest" />
</ItemGroup>

Console.WriteLine(L.Hello);
Install

Choose the package that fits your project.

Senlinz.Localization

Use this in consumer projects that need source generation from JSON.

dotnet add package Senlinz.Localization

Senlinz.Localization.Abstractions

Use this when you only need the shared runtime contracts without the generator package.

dotnet add package Senlinz.Localization.Abstractions
Quick start

Go from JSON to localized output in three clear steps.

1. Create your localization file

{
  "hello": "Hello",
  "sayHelloTo": "Hello {name}!",
  "statusReady": "Ready"
}

2. Register the file in your project

<ItemGroup>
  <AdditionalFiles Include="l.json" />
  <None Update="l.json" CopyToOutputDirectory="PreserveNewest" />
</ItemGroup>

3. Resolve localized text

var resolver = new LStringResolver(
    () => currentCulture,
    new EnResource(),
    new ZhResource());

Console.WriteLine(resolver[L.Hello]);
Console.WriteLine(resolver[L.SayHelloTo("World")]);
Use SenlinzLocalizationFile in your project file when you want a custom JSON file name instead of l.json.
Generated API

Understand what the generator creates for you.

L

Contains strongly typed accessors for each localization key.

LResource

Provides the generated base class for culture-specific overrides.

LString

Carries the key, fallback text, and runtime arguments.

LStringResolver

Resolves translated text from resources or an existing provider.

Placeholder parameters become method parameters

{
  "welcomeUser": "Welcome {userName}",
  "orderSummary": "Order {orderId} for {customerName}"
}
var message1 = L.WelcomeUser("Alice");
var message2 = L.OrderSummary("SO-001", "Alice");
Resource resolution

Use simple culture classes with predictable fallback behavior.

LResource gives each culture a generated base class. Override only the members that differ from the default JSON text. 

public sealed class EnResource : LResource
{
    public override string Culture => "en";

    protected override string Hello => "Hello";
}

public sealed class ZhResource : LResource
{
    public override string Culture => "zh";

    protected override string Hello => "你好";
}
  • If no culture resource exists, the JSON fallback text is returned.
  • If a resource exists but does not override a member, the JSON fallback text is returned.
  • Resources can be passed directly or through LResourceProvider.
Enum localization

Map enums to localization keys without extra conversion glue.

Generate a ToLString() extension with [LString]

[LString]
public enum UserType
{
    Teacher,
    Student
}

The default key pattern is <EnumName>_<MemberName>.

Customize enum member keys with [LStringKey]

[LString]
public enum UserType
{
    [LStringKey("teacher")]
    Teacher,

    [LStringKey("student")]
    Student
}

Use LStringAttribute when you also want to customize the separator.

Release workflow

Ship packages with the same workflow used in CI.

  1. Restore, build, test, and pack the solution.
  2. Create a version tag such as v1.1.0.
  3. The publish workflow uploads the generated .nupkg and optional .snupkg.
dotnet test Senlinz.Localization.slnx --configuration Release
dotnet pack Senlinz.Localization.slnx --configuration Release --output artifacts
