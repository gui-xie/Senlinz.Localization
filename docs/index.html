Core capabilities

Strongly typed localization APIs with flexible resource management.

Senlinz.Localization focuses on the parts teams care about most: maintainable strings, predictable generated APIs, and a runtime model that stays easy to reason about.

Generated API surface

Turn JSON keys into discoverable C# members.

Plain values become properties, placeholders become methods, and the generated names stay aligned with your localization source file.

  • hello → L.Hello
  • sayHelloTo → L.SayHelloTo(string name)
  • Generated types stay small and easy to navigate in IDEs
Resources

Culture-specific overrides, only where needed.

Implement one resource per culture and override only the values that differ from the default text.

Fallback behavior

Safe defaults for missing cultures and keys.

If a culture resource or member is missing, resolution falls back to the default text from JSON.

Enum localization

Bridge enums and localization keys cleanly.

Use attributes to map enum members into localized strings without repeating conversion logic.

Runtime contracts

Keep the runtime lightweight.

Use Senlinz.Localization.Abstractions when you only need shared contracts without pulling in the generator package.

Quick start

From JSON file to localized output in a few clear steps.

Use the default l.json convention or customize the file name with project properties.

  1. Create your localization file

    Define the source strings that will drive generated members.

    {
  "hello": "Hello",
  "sayHelloTo": "Hello {name}!",
  "statusReady": "Ready"
}

  2. Register it in the project

    Expose the file to the source generator and optionally copy it to output.

    <ItemGroup>
  <AdditionalFiles Include="l.json" />
  <None Update="l.json" CopyToOutputDirectory="PreserveNewest" />
</ItemGroup>

  3. Resolve text at runtime

    Pass resources directly or through a provider, then resolve using the current culture.

    var resolver = new LStringResolver(
    () => currentCulture,
    new EnResource(),
    new ZhResource());

Console.WriteLine(resolver[L.Hello]);

  • LStringResolver and LResourceProvider for runtime resolution
