Senlinz.Localization · v1.1.0

JSON-driven localization for .NET

Generate strongly typed localization APIs from JSON, implement culture-specific resources, and resolve translated text with a small runtime surface.

Why this package

  • Generate L accessors from l.json.
  • Generate LResource base types for per-culture overrides.
  • Resolve text through LStringResolver with fallback behavior.
  • Map enums to localization keys with [LString] and [LStringKey].

Install

dotnet add package Senlinz.Localization

Use Senlinz.Localization.Abstractions only if you need the shared runtime contracts without the source generator package.

Quick start

1. Add l.json

{
  "hello": "Hello",
  "sayHelloTo": "Hello {name}!",
  "statusReady": "Ready"
}

2. Register it in the project

<ItemGroup>
  <AdditionalFiles Include="l.json" />
  <None Update="l.json" CopyToOutputDirectory="PreserveNewest" />
</ItemGroup>

3. Use generated members

Console.WriteLine(L.Hello);
Console.WriteLine(L.SayHelloTo("World"));

4. Resolve localized text

var currentCulture = "zh";
var resolver = new LStringResolver(
    () => currentCulture,
    new EnResource(),
    new ZhResource());

Console.WriteLine(resolver[L.Hello]);

Resource resolution

Pass resources directly to LStringResolver for the common case, or pass an existing LResourceProvider if your application already owns one. 

var provider = new LResourceProvider(
    new EnResource(),
    new ZhResource());

var resolver = new LStringResolver(
    () => currentCulture,
    provider);
  • If a culture resource is missing, the default l.json text is used.
  • If a key is missing in a culture resource, the default text is used.
  • Resources are cached per resolver instance and culture.

Enum localization

[LString]
public enum UserType
{
    Teacher,
    Student
}

var text = UserType.Student.ToLString();
Console.WriteLine(resolver[text]);

Use [LStringKey] when an enum member should map to an existing localization key while preserving the enum prefix behavior.

Release workflow

  1. Run validation and ensure CI passes.
  2. Create a version tag such as v1.1.0.
  3. The NuGet publish workflow restores, builds, tests, packs, and uploads artifacts.
  4. The generated .nupkg and optional .snupkg packages are published to NuGet.
