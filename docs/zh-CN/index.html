Senlinz.Localization · v1.1.0

.NET 的 JSON 本地化源码生成器

从 JSON 生成强类型本地化 API，为不同语言实现资源类，并通过极小的运行时抽象完成文本解析。

English 中文

它能做什么

  • 从 l.json 生成 L 访问器。
  • 生成适合多语言覆写的 LResource 基类。
  • 通过 LStringResolver 与回退机制解析文本。
  • 通过 [LString] 和 [LStringKey] 将枚举映射到本地化键。

安装

dotnet add package Senlinz.Localization

如果你只需要运行时契约，不需要源码生成器，可以单独安装 Senlinz.Localization.Abstractions。

快速开始

1. 添加 l.json

{
  "hello": "Hello",
  "sayHelloTo": "Hello {name}!",
  "statusReady": "Ready"
}

2. 在项目中注册

<ItemGroup>
  <AdditionalFiles Include="l.json" />
  <None Update="l.json" CopyToOutputDirectory="PreserveNewest" />
</ItemGroup>

3. 使用生成代码

Console.WriteLine(L.Hello);
Console.WriteLine(L.SayHelloTo("World"));

4. 解析本地化文本

var currentCulture = "zh";
var resolver = new LStringResolver(
    () => currentCulture,
    new EnResource(),
    new ZhResource());

Console.WriteLine(resolver[L.Hello]);

资源解析

常见场景下，直接把资源实例传给 LStringResolver 即可； 如果你的应用已经维护了 LResourceProvider，也可以直接复用。 

var provider = new LResourceProvider(
    new EnResource(),
    new ZhResource());

var resolver = new LStringResolver(
    () => currentCulture,
    provider);
  • 语言资源缺失时，会回退到 l.json 中的默认文本。
  • 资源中缺少某个键时，也会回退到默认文本。
  • 资源字典会按解析器实例和语言分别缓存。

枚举本地化

[LString]
public enum UserType
{
    Teacher,
    Student
}

var text = UserType.Student.ToLString();
Console.WriteLine(resolver[text]);

当枚举成员需要映射到现有键时，请使用 [LStringKey]， 这样仍然可以保留默认的枚举前缀规则。

发布流程

  1. 先运行校验并确保 CI 通过。
  2. 创建类似 v1.1.0 的版本标签。
  3. NuGet 发布工作流会自动执行 restore、build、test、pack 并上传制品。
  4. 生成的 .nupkg 与可选的 .snupkg 会被发布到 NuGet。

GitHub Pages

仓库现在带有专门的 Pages 工作流，会在推送到 main 或手动触发时， 自动发布 /docs 目录中的静态文档站点。

只需在仓库设置里将 GitHub Pages 来源切换为 GitHub Actions 即可启用。
