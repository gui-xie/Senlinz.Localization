概览

让 GitHub Pages 更像真正的开发者文档站。

Senlinz.Localization 是一个基于 JSON 的 .NET 本地化源码生成器。它会把本地化键转换成强类型 API， 支持语言资源覆写，并在资源缺失时回退到默认 JSON 文本。

强类型访问器 按语言覆写资源 轻量运行时抽象
适合场景

让本地化配置更贴近代码

  • 从 l.json 直接生成 L 成员。
  • 只覆写真正变化的语言值。
  • 缺失资源或缺失键时，自动回退到默认文本。
5 分钟接入 
dotnet add package Senlinz.Localization

<ItemGroup>
  <AdditionalFiles Include="l.json" />
  <None Update="l.json" CopyToOutputDirectory="PreserveNewest" />
</ItemGroup>

Console.WriteLine(L.Hello);
安装

按项目需求选择对应包。

Senlinz.Localization

用于需要根据 JSON 生成代码的消费项目。

dotnet add package Senlinz.Localization

Senlinz.Localization.Abstractions

用于只需要共享运行时契约、但不需要源码生成器的场景。

dotnet add package Senlinz.Localization.Abstractions
快速开始

从 JSON 到本地化输出，只需要三个步骤。

1. 创建本地化文件

{
  "hello": "Hello",
  "sayHelloTo": "Hello {name}!",
  "statusReady": "Ready"
}

2. 在项目里注册文件

<ItemGroup>
  <AdditionalFiles Include="l.json" />
  <None Update="l.json" CopyToOutputDirectory="PreserveNewest" />
</ItemGroup>

3. 运行时解析文本

var resolver = new LStringResolver(
    () => currentCulture,
    new EnResource(),
    new ZhResource());

Console.WriteLine(resolver[L.Hello]);
Console.WriteLine(resolver[L.SayHelloTo("世界")]);
如果你不想使用 l.json，可以在项目文件里通过 SenlinzLocalizationFile 指定自定义文件名。
生成 API

了解生成器会自动为你准备什么。

L

为每个本地化键提供强类型访问入口。

LResource

提供语言资源覆写的生成基类。

LString

携带键名、默认文本以及运行时参数。

LStringResolver

从资源类或资源提供器中解析最终文本。

占位符会自动变成方法参数

{
  "welcomeUser": "Welcome {userName}",
  "orderSummary": "Order {orderId} for {customerName}"
}
var message1 = L.WelcomeUser("Alice");
var message2 = L.OrderSummary("SO-001", "Alice");
资源解析

用简单的语言资源类获得稳定回退行为。

LResource 会为每种语言提供生成基类。你只需要覆写与默认 JSON 文本不同的成员。 

public sealed class EnResource : LResource
{
    public override string Culture => "en";

    protected override string Hello => "Hello";
}

public sealed class ZhResource : LResource
{
    public override string Culture => "zh";

    protected override string Hello => "你好";
}
  • 当前语言没有资源时，返回 JSON 默认文本。
  • 资源存在但没有覆写某个成员时，也会回退到 JSON 默认文本。
  • 资源既可以直接传给 LStringResolver，也可以先封装成 LResourceProvider。
枚举本地化

让枚举直接映射到本地化键，不需要额外转换层。

通过 [LString] 生成 ToLString() 扩展

[LString]
public enum UserType
{
    Teacher,
    Student
}

默认键格式为 <EnumName>_<MemberName>。

通过 [LStringKey] 自定义成员键

[LString]
public enum UserType
{
    [LStringKey("teacher")]
    Teacher,

    [LStringKey("student")]
    Student
}

如果还需要自定义分隔符，可以配合 LStringAttribute 使用。

包发布

沿用 CI 中的验证流程来发布 NuGet 包。

  1. 先执行 restore、build、test、pack。
  2. 创建类似 v1.1.0 的版本标签。
  3. 发布工作流会上传生成的 .nupkg 与可选 .snupkg。
dotnet test Senlinz.Localization.slnx --configuration Release
dotnet pack Senlinz.Localization.slnx --configuration Release --output artifacts
GitHub Pages

直接从 /docs 发布静态文档站点。

仓库已经内置 Pages 工作流，会在推送到 main 或手动触发时部署 docs/ 下的内容。

  • 在仓库设置中将 GitHub Pages 来源设置为 GitHub Actions。
  • 保持 docs/index.html、docs/zh-CN/index.html 与 docs/assets/styles.css 一起提交。
  • 通过顶部语言切换保持中英文文档风格一致。
